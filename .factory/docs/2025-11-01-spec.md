## カフェオーダーシステム - プロジェクト初期構成計画

### 1. **プロジェクト構造の作成**
```
ital-cafe/
├── frontend/                 # Next.js フロントエンド
│   ├── src/
│   │   ├── app/             # App Router
│   │   ├── components/      # 共通コンポーネント
│   │   ├── lib/             # ユーティリティ
│   │   ├── hooks/           # カスタムフック
│   │   └── styles/          # グローバルスタイル
│   └── public/
├── backend/                  # Hono バックエンド
│   ├── src/
│   │   ├── routes/          # APIルート
│   │   ├── middlewares/     # ミドルウェア
│   │   ├── services/        # ビジネスロジック
│   │   └── db/              # データベース関連
│   └── prisma/              # Prisma設定
├── docker/                   # Docker設定
│   ├── frontend/
│   ├── backend/
│   └── postgres/
├── docker-compose.yml       # Docker Compose設定
└── .env.example             # 環境変数テンプレート
```

### 2. **Docker環境の設定**
- **Frontend**: Next.js 15+ (Node.js 20)
- **Backend**: Hono (Node.js 20)
- **Database**: PostgreSQL 16
- **開発環境**: Hot Reload対応

### 3. **フロントエンド初期構成**
- Next.js 15 (App Router)
- TypeScript
- Tailwind CSS
- shadcn/ui
- React Query (TanStack Query)
- Storybook設定

### 4. **バックエンド初期構成**
- Hono.js
- Prisma ORM
- TypeScript
- JWT認証
- Zod (バリデーション)

### 5. **データベース設計**
要件定義書に基づいたPrismaスキーマ作成：
- Product (単品/セット商品)
- SetComponent (セット構成品)
- Topping
- Order/OrderItem
- SessionToken
- TaxRateSchedule
- User/Table

### 6. **初期実装の優先順位**
1. Docker環境の構築
2. Next.js/Hono基本セットアップ
3. Prismaスキーマ定義
4. 認証・セッション管理
5. UI基本コンポーネント（デザインHTML参考）

### 7. **開発規約の適用**
- 既存のデザインHTML（色・レイアウト）を忠実に再現
- コンポーネント化と再利用性を重視
- Storybookで全コンポーネントを管理
- 日本語レスポンス対応
